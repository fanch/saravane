Why should you contribute ?

Today NuTyX have about 1100 applications in binaries available. It's a lot and not because every days are new programs release on the net under open source licence (or not)

It's clear, you will never have the chance to see all thoses programs ready to be installed on NuTyX. Just with the web site sourceforge.net, we can find more then 5000 applications for all platforms. If you found the application of yours dreams, you will have in eather way to compile it, install it and make it working on NuTyX. That why we will ask you to contribute a little bit. It's you who will build the package... if you want of course.

Compiling an application is often something like:

  • Find the application that doen't exist on NuTyX
  • Download the sources of the application
  • Check the MD5SUM (strongly recommended but not always possible)
  • Extract sources in a working directory
  • Enter in this working directory
  • Check that all the dependencies exists in NuTyX so that the compilation endup with succes
  • Type the three magic commands (not always the case)
./configure --prefix=/usr
make
make install

That's it you new program is compiled and install on your NuTyX.

That's one way to go... but as you can see they are a few disadvantages:

  • Impossible to know which files are installed
  • Impossible get information about the installed package like name, version, description, etc...
  • Impossible to share the application with another installation of NuTyX
  • This method is quiete heavy because need to be done again when they will be an update available for example.
  • Your NuTyX is not really clean anymore since you have add files that are not track by the package manager.

    •  

    NuTyX have nice tools to help you greatly the integration of new programs. You will be then able to track them, maintain them. The package manager will treat them like other NuTyX packages.
