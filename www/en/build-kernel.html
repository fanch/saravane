Compile and install your own kernel

Introduction

This topic can be apply to any distribution, not only NuTyX.

If you decide to compile the kernel yourself, this topic will explain you how to compile and install a cutomize kernel.

They are a lot of raisons to compile your own kernel. One main reason could be to reduce as much as possible it's size and optimize for it's own hardware would be a good reason.

Actions to take

Here is a summary of what we have to do. It's good to know that the steps 4 and 5 can be done as many time as we want (in case of a fail boot of the new compiled kernel). So we have to:

  1. Download the sources of the kernel from the URL http://kernel.org
  2. Extracts the sources from the downloaded kernel
  3. Change to the sources directory of the kernel and type in the magics commands to configure the options we want
  4. Configure the compilation options
  5. Compile the kernel
  6. Create a new entry in the grub.cfg or menu.lst file

Download the sources of the kernel

To be able to compile a kernel, we need the sources of a kernel version.At the writing time of this article, the version 3.17.4 is available. We will then take the chance to compile it...

Open a terminal and type the command:

wget http://www.kernel.org/pub/linux/kernel/v3.x/linux-3.17.4.tar.xz

Extract the sources from the downloaded kernel

We extract the sources from our new downloaded kernel:

tar xf linux-3.17.4.tar.xz

After a few minutes, the prompt will be available and the sources are in place. To be able to compile the kernel, we need to be in the directory of the kernel and then we can type the magics commands. Type in your terminal:

Go to the sources directory

cd linux-3.17.4

We are now ready to launch the magics commands

As a start, I recommend you to consult all the possible options in the compilation:

make help|more

It will give you ALL the possibles options to compile the kernel and it's modules.

Configure the compilations options

The team of LFS recommand to deactivate the variable LC_ALL :

make LC_ALL= menuconfig

Once you did all your selections, you choose the option Exit until you are back at the prompt.

A new message will ask you if you want to save your configuration, choose the default option Yes.

A nice message in your terminal will appear and tell you :

#
# configuration written to .config
#


* End of the configuration.
* Execute 'make' to start the build or try 'make help'

Compile the kernel

The compilation of your kernel can start. Type in your terminal:

make -j4

The compilation starts, if you have more or less processors, adjust the value -j4 to the number of cores you have

A few minutes (or hours) later, ... You have a kernel ready to used.

Kernel: arch/x86/boot/bzImage is ready  (#1)

Create a new entry in the grub.cfg or menu.lst file

We just have to add the right entry into the configuration files grub.cfg of grub 2.0 or eventuelly in menu.lst of grub 0.97.

1. /boot/grub/grub.cfg

 ....
 
menuentry "NuTyX avec kernel experimental sur /dev/sda2" {
  set root=(hd0,2)
  linux /home/thierry/linux-3.17.4/arch/x86/boot/bzImage root=/dev/sda2 ro quiet
}

2. /boot/grub/menu.lst

...
# (3) NuTyX sur /dev/sda2
title NuTyX avec kernel experimental sur /dev/sda2
root (hd0,1)
kernel /home/thierry/linux-3.17.4/arch/x86/boot/bzImage root=/dev/sda2 ro quiet

And that's it, this way, you will be still able to choose at boot time on which kernel your NuTyX should be running.

Last but not least, to experiment other options of your working (or not yet) new kernel, it's very simple. You just type again the 4 magic commands:

cd linux-3.17.4
make menuconfig
make -j4
sudo make modules_install

And that's all
